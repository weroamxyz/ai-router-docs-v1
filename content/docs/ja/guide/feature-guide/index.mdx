ComputeVault は統合 LLM ゲートウェイプラットフォームです。複数の AI プロバイダーの API を標準的な OpenAI 互換インターフェースに集約し、1 つのエンドポイントから数十種類のモデルにアクセスできます。本ガイドはロール別にユーザーガイドと管理者ガイドに分かれています。

![ホームページ](/assets/guide/home.png)

## ロール説明

プラットフォームには権限レベルの異なる 3 つのロールがあります。

### 一般ユーザー

登録後のデフォルトロール。トークン作成、API 呼び出し、使用量確認、チャージとサブスクリプションが可能です。

### 管理者（Admin）

Root により昇格されたロール。ユーザー権限に加え、チャンネル、ユーザー、換金コード、ログ、モデル、グループの管理が可能です。

### スーパー管理者（Root）

最高権限レベル。管理者権限に加え、システム設定、カスタム OAuth、パフォーマンス監視が可能です。

## ユーザーガイド

すべてのログインユーザー向け。登録から日常利用までの完全なフローをカバーします。

### 登録とログイン

パスワードログインおよび GitHub、Discord などの OAuth ログインに対応。

- パスワードログイン
- サードパーティ OAuth ログイン
- パスワードリセット
- 新規ユーザー登録

[詳細ドキュメントを見る →](/ja/docs/guide/feature-guide/user/auth)

### 個人設定

アカウント情報の管理、2FA の有効化、Passkey のパスワードレスログイン設定。

- ユーザー名とパスワードの変更
- メールアドレスの登録
- 二要素認証（2FA）の有効化
- Passkey のパスワードレスログイン登録
- サードパーティアカウントの連携

[詳細ドキュメントを見る →](/ja/docs/guide/feature-guide/user/personal-setting)

### トークン管理

API 認証情報の作成と管理。クォータ制限、モデル制限、IP 許可リストに対応。

- 新しいトークンの作成
- トークン権限の設定
- クォータ制限の設定
- モデル制限の設定
- IP 許可リストの設定

[詳細ドキュメントを見る →](/ja/docs/guide/feature-guide/user/token)

### API の使い方

OpenAI、Claude、Gemini 形式に対応。オンラインテスト用 Playground を内蔵。

- Playground オンラインテスト
- API アドレスの取得
- コード例（Python、cURL）
- サポートされている API エンドポイント

[詳細ドキュメントを見る →](/ja/docs/guide/feature-guide/user/api)

### 使用ログ

各 API 呼び出しのモデル、トークン消費、クォータ控除の詳細を確認。

- 呼び出しログの確認
- レコードの検索とフィルタリング
- データ統計チャート
- クォータ消費分析

[詳細ドキュメントを見る →](/ja/docs/guide/feature-guide/user/log)

### クォータとチャージ

換金コード、EPay、Stripe などでチャージ。紹介報酬制度あり。

- オンライン決済チャージ
- 換金コードチャージ
- 紹介報酬プログラム
- チャージ履歴の確認

[詳細ドキュメントを見る →](/ja/docs/guide/feature-guide/user/topup)

### サブスクリプションプラン

安定した使用量ニーズを持つユーザー向けの定期購読パッケージ。

- サブスクリプションプランの確認
- サブスクリプションの購入
- 自動更新の管理

[詳細ドキュメントを見る →](/ja/docs/guide/feature-guide/user/subscription)

### 料金

全モデルの料金と課金情報を確認。

- モデル料金の確認
- 特定モデルの検索
- 課金ルールの理解

[詳細ドキュメントを見る →](/ja/docs/guide/feature-guide/user/pricing)

## 管理者ガイド

Admin および Root ロール向け。プラットフォーム運営とシステム設定の全機能をカバーします。

### チャンネル管理

AI プロバイダーチャンネルの設定。マルチキーポーリング、優先度、自動無効化に対応。

- 新しいチャンネルの追加
- チャンネルパラメータの設定
- チャンネルテスト
- 一括操作
- マルチキーモード設定

[詳細ドキュメントを見る →](/ja/docs/guide/feature-guide/admin/channel)

### ユーザー管理

全ユーザーの確認と管理。ロール、グループ、クォータの変更、アカウントの有効化・無効化。

- ユーザーリストの確認
- ユーザー情報の編集
- ユーザーロールの変更
- ユーザークォータの調整
- ユーザーの検索

[詳細ドキュメントを見る →](/ja/docs/guide/feature-guide/admin/user)

### 換金コード管理

クォータ換金コードの一括生成と管理。

- 換金コードの一括生成
- コード額面の設定
- コードのエクスポート
- 使用状況の確認

[詳細ドキュメントを見る →](/ja/docs/guide/feature-guide/admin/redemption)

### ログ管理

プラットフォーム全体の API 呼び出しログとユーザー操作記録を確認。

- 全ログの確認
- 高度なフィルタリング
- データ統計分析

[詳細ドキュメントを見る →](/ja/docs/guide/feature-guide/admin/log)

### サブスクリプション管理

サブスクリプションパッケージの作成と管理。

- サブスクリプションプランの作成
- プラン情報の編集
- サブスクライブユーザーの管理

[詳細ドキュメントを見る →](/ja/docs/guide/feature-guide/admin/subscription)

### モデル管理

モデルの料金と可用性を設定。

- モデル倍率の設定
- モデルの有効化・無効化
- 料金の一括変更

[詳細ドキュメントを見る →](/ja/docs/guide/feature-guide/admin/model)

### グループ管理

差別化された課金とチャンネル割り当てのためのユーザーグループを作成。

- ユーザーグループの作成
- グループ倍率の設定
- チャンネルの割り当て

[詳細ドキュメントを見る →](/ja/docs/guide/feature-guide/admin/group)

### システム設定（Root 専用）

Root 専用グローバル設定センター。サイト情報、課金ルール、支払い、OAuth などを設定。

- サイト基本情報
- 課金ルール設定
- 支払い方法設定
- メールサービス設定
- 登録とログイン設定

[詳細ドキュメントを見る →](/ja/docs/guide/feature-guide/admin/system-setting)

### カスタム OAuth（Root 専用）

カスタム OAuth ログインプロバイダーの設定。

- OAuth プロバイダーの追加
- OAuth パラメータの設定
- OAuth ログインのテスト

[詳細ドキュメントを見る →](/ja/docs/guide/feature-guide/admin/custom-oauth)

### パフォーマンス監視（Root 専用）

システムパフォーマンス指標と運用状況を確認。

- システム負荷の確認
- チャンネルパフォーマンスの監視
- レスポンスタイムの分析

[詳細ドキュメントを見る →](/ja/docs/guide/feature-guide/admin/performance)

### ドキュメント設定（Root 専用）

ドキュメントサイトの表示内容を設定。

- ドキュメントコンテンツのカスタマイズ
- ドキュメントリンクの設定

[詳細ドキュメントを見る →](/ja/docs/guide/feature-guide/admin/docs-config)
