## 主要機能

### 💬 マルチセッションチャット
- **マルチセッション + 独立したコンテキスト** - 複数のチャットセッションを同時に開くことができ、各セッションは独立したコンテキスト記憶を持ちます
- **ローカルストレージ** - すべての会話データはローカルのSQLiteデータベースに保存され、失われることはありません

### 🤖 マルチモデルサポート
- **マルチプラットフォームサポート** - Gemini、OpenAI、Claude、Qwenなどの主要モデルをサポートし、柔軟に切り替え可能です
- **ローカルモデルサポート** - Ollama、LM Studioなどのローカルモデルデプロイメントをサポートします

### 🤝 マルチエージェントモード
- **複数のAIエージェントを同時に実行** - 複数のAIエージェント（Gemini CLI、Claude Code、Codex、OpenCode、Qwen Code、Goose CLI、Auggieなど）を同時に実行できます
- **MCPによる統合管理** - Model Context Protocol (MCP) を介してすべてのエージェントを統合的に管理および設定し、操作プロセスを簡素化します
- **スキル設定** - 異なるエージェントに専用のスキルを設定し、エージェントの機能を拡張できます
- **アシスタントのカスタマイズ** - アシスタント設定をカスタマイズし、パーソナライズされたAIワークフローを作成できます
- **独立した設定** - 各エージェントは独立して設定および使用でき、互いに干渉しません
- **柔軟な切り替え** - 異なるエージェント間で柔軟に切り替え、さまざまなシナリオのニーズに対応します

### 🗂️ ファイル管理
- **ファイルツリー表示 + ドラッグ＆ドロップアップロード** - フォルダのようにファイルを閲覧し、ファイルやフォルダをドラッグ＆ドロップで一括インポートできます
- **スマート整理** - AIがフォルダの整理を支援し、自動的に分類できます

### 📄 プレビューパネル
- **9種類以上の形式プレビュー** - PDF、Word、Excel、PPT、コード、Markdown、画像などの形式をサポートします
- **リアルタイム追跡 + 編集可能** - ファイルの変更を自動的に追跡し、Markdown、コード、HTMLのリアルタイム編集とデバッグをサポートします

### 🎨 AI画像生成と編集
- **スマート画像生成** - Gemini 2.5 Flash Image Preview、Nano、Bananaなど、さまざまな画像生成モデルをサポートします
- **画像認識と編集** - AIを活用した画像分析および編集機能

### 🌐 マルチチャネルアクセス
- **WebUIリモートアクセス** - ブラウザを介してネットワーク上の任意のデバイスからアクセスでき、モバイルデバイスもサポートします
- **Telegram連携** - Telegramボットを介したインタラクションをサポートします
- **Feishu連携** - Feishuを介したアクセスとインタラクションをサポートします
- **ローカルデータセキュリティ** - すべてのデータはローカルのSQLiteデータベースに保存され、サーバーデプロイメントに適しています

## NewAPI接続方法

### パラメータ入力

プロバイダータイプ：NewAPIがサポートするタイプ
APIキー：NewAPIで取得
APIアドレス：NewAPIサイトアドレス（例：`https://<your-newapi-domain>/v1`）

ご自身でデプロイしたUnode、またはサービスプロバイダーが合法的なアップストリーム認証とコンプライアンス義務を持つUnodeサービスを使用してください。出所の不明なAPIアドレスやキーを本番環境に接続しないでください。

### 設定手順

1. **NewAPIでAPIキーをコピー**
   ![APIキーをコピー](/assets/aionui/copy_apikey.png)

2. **AionUi設定を開く**
   - AionUiで設定ページに移動します
   - モデル設定タブを見つけます
   - 「モデルを追加」をクリックします
   ![設定を開く](/assets/aionui/add-model-1.png)

3. **新しいプロバイダーを追加**
   - 「モデルを追加」をクリックします
   - NewAPIを選択します
   ![NewAPIプロバイダーを追加](/assets/aionui/newapi_provider.png)

4. **API情報を設定**
   - APIアドレス：NewAPIサイトアドレスを入力します（形式：`https://<your-newapi-domain>/v1`）
   - APIキー：NewAPIコンソールからコピーしたAPIキーを貼り付けます

5. **モデルを追加**
   - ドロップダウンから追加したいモデルを選択します
   - モデル名はNewAPIで設定されているモデル名と一致している必要があります
   - 適切なリクエストプロトコルを選択します

6. **使用を開始**
   - チャットページに戻ります
   - 設定済みのNewAPIモデルを選択して会話を開始します
